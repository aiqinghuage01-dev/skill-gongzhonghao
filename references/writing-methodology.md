# 公众号写作方法论（学员版完整版）

> 本方法论对任何行业的老板都适用。写作时所有"我"的细节（资历/数据/案例）
> 必须从老板本地内容库（`~/Desktop/我的内容库（勿动）/01-黄金五环` +
> `03-人设故事`）提取，禁止 AI 编造。

> 本文件是**公众号专用**写作方法论。底层逻辑来自15个通用内容技能文件（钩子、留存、植入、人设等），
> 但所有规则已针对公众号的阅读场景、分发机制、视觉呈现做了专门适配。
> 公众号不是"把短视频脚本贴上去"，而是一种独立的内容形态。

## 零、公众号适配层

公众号文章和短视频有完全不同的生存规则。下面这些规则优先级最高，覆盖后续所有章节中与短视频思维冲突的地方。

### 0.1 打开率 = 标题 × 封面

短视频靠前3秒钩人，公众号靠**标题**钩人。用户在订阅消息列表里只能看到标题和封面，点不点全靠这两样。

**标题规则：**
- 字数控制在 **15-25字**，太短没信息量，太长折行难看
- 必须包含一个**情绪触发词**（别、千万别、真相、翻车、坑、血亏、逆袭、打脸）
- 必须包含一个**身份锚点**（老板、实体人、开店的、做生意的）
- 可选：包含一个**悬念缺口**（为什么、怎么做到的、竟然、没想到）

**标题模板（可混搭）：**
- 结论前置型：`别瞎学AI，实体老板最该做的只有这三件事`
- 反常识型：`越努力的老板，越容易被AI割韭菜`
- 数字清单型：`实体老板用AI的5个致命误区，第3个最常见`
- 故事悬念型：`一个餐饮老板花了3万学AI，最后只用了一招就回本`
- 热点借势型：`XX事件背后，实体老板必须看懂的一个信号`
- 对比冲突型：`同样用AI，为什么他月入百万你月亏三万？`

**封面规则：**
- 大字 + 纯色/渐变背景，在手机小图下也能看清
- 标题中最有冲击力的2-4个字作为封面主标题
- 副标题补充语境
- 不要用密集的图片做封面，订阅列表里缩成小图后什么都看不清

### 0.2 分发靠转发，不靠算法

短视频靠平台算法推荐，公众号靠**读者主动转发到朋友圈/群聊**。这意味着：

**写作时必须考虑：读者转发的动机是什么？**

读者转发的5种心理：
1. **表达自我**："这就是我想说的" → 文章要替读者说出心里话
2. **社交货币**："我知道一个别人不知道的事" → 文章要有信息增量
3. **利他心理**："这对我朋友有用" → 文章要有明确的实用价值
4. **身份认同**："这代表我是什么样的人" → 文章要能让老板群体产生归属感
5. **情绪共鸣**："说得太对了" → 文章要击中痛点

**转发触发设计（至少埋1-2个）：**
- 在文章中间或结尾放一句"转发级金句"——读者看到就想截图或分享的话
- 句式：`这句话送给每一个还在硬扛的实体老板。`
- 句式：`如果你身边有老板还在纠结要不要做AI，把这篇转给他。`
- 句式：`看懂这三条，至少帮你省X万学费。`

**"在看"触发：**
- 文章末尾用一句话引导"如果觉得有用，点个在看，让更多老板看到"
- 不用求、不用卑微，用"让更多人看到"作为理由，读者更愿意点

### 0.3 阅读节奏 ≠ 听觉节奏

短视频靠语气、停顿、表情来控制节奏。公众号文章是纯视觉消费，读者在手机上竖着滑，每一屏大概能看200-300字。

**视觉节奏规则：**
- **每3-5段插一个视觉断点**：高亮框、引用块、加粗金句、分隔线、emoji小标题
- **段落不超过4行**（手机屏幕上4行以上就成文字墙）
- **关键观点单独成段，加粗或放在引用框里**，让扫读的人也能抓到重点
- **不要连续5段以上纯文字**，读者会疲劳划走

**层级感设计：**
- 用大标题、小标题、数字编号建立清晰层级
- 读者应该能通过扫标题就知道全文在讲什么
- 每个大段都有一个"一句话总结"放在开头或结尾

### 0.4 深度是公众号的护城河

短视频最多讲3分钟，公众号可以讲3000字。利用好这个优势：

- **案例可以展开**：短视频只能一句话带过的案例，公众号可以讲来龙去脉
- **逻辑可以完整**：短视频只能给结论，公众号可以把推导过程讲清楚
- **数据可以呈现**：对比表格、数字列举、前后对比，这些在短视频里做不了
- **但不要滥用深度**：深≠长。每一段都必须服务于"让老板看懂且能执行"

### 0.5 CTA路径适配

短视频的CTA：`点头像 → 进直播间 → 加助教`
公众号的CTA路径不同：

**主CTA路径（选一个）：**
- `关注公众号 → 回复关键词 → 领取资料/进群`
- `点击"阅读原文" → 落地页/报名页`
- `扫码添加微信 → 1v1沟通`（需要文章里有二维码图片）
- `文末留言区互动 → 评论区置顶引导`

**CTA语气（继承低压原则）：**
- `想要这套清单的老板，公众号回复"清单"，我发你。`
- `如果你也在做实体，欢迎在留言区聊聊你的行业。`
- `下一篇我会详细讲第3步怎么落地，先关注别错过。`

不要用：`立即报名！仅剩X个名额！`

### 0.6 公众号文章的独特优势（要发挥的）

1. **长尾流量**：一篇好文章可以被反复转发几个月，不像短视频48小时后基本没流量
2. **搜索入口**：微信搜一搜会收录公众号文章，标题里的关键词很重要
3. **信任建设**：读完一篇3000字深度文章建立的信任感，远超一条60秒短视频
4. **私域沉淀**：公众号粉丝是你的，不受平台算法波动影响
5. **内容复用**：一篇公众号文章可以拆成5-10条短视频脚本

**搜一搜优化：**
- 标题中包含用户会搜索的关键词（"实体老板""AI获客""短视频获客"）
- 文章开头200字要自然包含核心关键词
- 这不是SEO堆砌，而是用老板真正会搜的词来写

## 一、文章骨架结构

来源：peer-model.md, peer-copy-mechanism.md

### 7步骨架（按顺序执行）

**第1步：爆点开场（3句）**
- 第1句：直接态度（强判断），不做背景铺垫
- 第2句：一句画面化细节（让人脑补现场）
- 第3句：一句"这事和你有关"的拉近

示例骨架：
- "看完这个动作，我脑子里只剩四个字：____。"
- "他以为在自证，实际是在____。"
- "老板们别看热闹，这就是你明年的经营难题。"

**第2步：角色冲突（2-3段）**
- 角色A：实体强者，但线上表达弱
- 角色B：流量玩家，擅长情绪与议题设置
- 冲突核心：不是事实本身，而是"谁定义公众感受"

**第3步：理论锚点（1段）**
- 引入一个理论名词做"认知抓手"
- 理论只讲结论，不做学术展开
- 句式：`这就是典型的____：现实里的强者，线上未必是强者。`

**第4步：反转升维（1-2段）**
- 从"谁赢谁输"转成"老板该怎么活"
- 用"你以为…其实…"句式连续转折

**第5步：三条建议（固定）**
1. 表达姿态建议（去爹味，少辩对错，多给证据）
2. 经营动作建议（产品、服务、透明度、反馈机制）
3. 能力系统建议（引出业务：AI内容中台/私域/获客系统）

**第5.5步：业务预埋链**（见第四章详述）

**第5.6步：人设预埋链**（见第三章详述）

**第6步：结尾（2句）**
- 句1：金句结论
- 句2：行动号召（关注/私信/领取方法）

### 用户为什么愿意听下去（核心机制）

1. 高冲突判词开场：先给结论，不先讲背景
2. 连续推进设计：每隔几段就抛一个新判断，防掉线
3. 画面化比喻：电影、人物、场景让抽象观点可视化
4. 站队张力：天然引发评论区参与和二次传播

### 可复制 vs 不可照抄

可复制：节奏、推进、预埋链、口语听感
不可照抄：人身攻击、绝对化标签、羞辱式表达

## 二、开场与留存引擎

来源：hook-library.md, retention-engine-playbook.md

### 黄金三秒钩子库

1. `看完这件事，我脑子里只剩四个字：____。`
2. `这不是翻红，这是一次信任重估。`
3. `你以为这是明星新闻，其实是老板生存题。`
4. `同样是热点，有人吃到流量，有人吃到反噬。`
5. `这波最值钱的，不是销量，是解释权。`

使用规则：开场三句内必须有"判断+画面+关联老板"。

### 三层留存结构

**层1：黄金三秒（抢停留）**
- 用一个高冲突判断开场，不做背景铺垫

**层2：黄金三十秒（抢继续看）**
- 给一个具体画面或细节（谁、什么时候、说了什么）
- 立刻抛第二个判断，形成"连续推进"

**层3：全文留存（防掉线）**
- 每300-500字必须有一个推进动作：
  - 新冲突
  - 新类比
  - 新反转（你以为/其实）
  - 新问题（老板该怎么办）

### 口语节奏模板

- 长句只负责信息，短句负责节奏
- 每段建议2-4行，避免大段讲课感
- 每4-6段至少1个反问句，拉回注意力

### 降说教模板

- 说教句：`你必须马上改变。` → 改写：`这一步很多老板都会拖，但越拖成本越高。`
- 说教句：`你这样做就是错的。` → 改写：`这套做法在实体场景能跑，在舆论场景容易失效。`

## 三、人设构建方法

来源：persona-positioning-playbook.md, peer-model.md 5.6节

### 人设预埋链（前40%必须完成，不可缺项）

1. **资历点到即止**
   - `我在科技圈待了10年，做流量做了5年，见过太多老板在这一步踩坑。`
   - 只用1-2句交代年限和经历，不做履历朗读

2. **阵营站位清晰**
   - `我不是来评判谁高谁低，我是替一线老板拆解可执行的动作。`
   - 明确"我替老板说话"，快速建立同一阵营

3. **双边理解**
   - `实体老板的难我懂，流量场的规则我也懂。`
   - 同时承认两边，形成"翻译官+教练"角色

4. **价值承诺（反自夸）**
   - `说这些不是为了证明我多厉害，是为了让你少走弯路。`
   - 资历出现后立刻回到方法，不停留在自夸

5. **身段控制**
   - 避免"教训式"口吻，保持"并肩式"表达
   - `你先把这三步做完，再谈放大。`

### 人设禁区

- 连续三段都在讲自己经历
- "我早就说过/你们都不懂"的高位口吻
- 用羞辱词制造优越感
- 只讲观点不给动作

### 人设构建检查

1. 前20%是否完成"资历+阵营"？
2. 前40%是否完成"懂两边+反自夸"？
3. 是否至少给出3条可执行动作？
4. 是否让用户感到"他在帮我"，而不是"他在教训我"？

## 四、业务植入方法

来源：business-integration-playbook.md, soft-sell-library.md, person-controversy-business-persona-injection.md

### 业务植入六步法（必须按顺序）

1. **资历预埋**
   - 示例格式：`我做了 {N} 年 {行业}，{处理过的具体业务量}，见过太多同类翻车。`
   - 从老板 01-黄金五环 的"经历"字段取真实数字/年头/业务量填，**不要编**
   - 作用：先建立"我有资格说"

2. **立场预埋**
   - `我不是来吃瓜站队，我是替老板拆解风险。`
   - 作用：从喷子区拉到顾问区

3. **双边理解预埋**
   - `我理解实体老板的难，也看懂流量玩家的打法。`
   - 作用：建立"中立且懂行"的可信人设

4. **干货先行预埋**
   - `先给你三条能落地的动作。`
   - 作用：先给价值，再提业务，降低销售防御

5. **工具化植入**
   - `第3条需要系统化能力（内容/IP/直播/私域），这正是我们在做的。`
   - 作用：业务包装成"执行工具"而非广告

6. **低压CTA**
   - `想要模板/清单，私信关键词。`
   - 作用：降低动作门槛，承接线索而不逼单

### 低压种草句库

**预埋资历：**
- `我这几年一直在陪实体老板做流量和转化，见过太多同类翻车。`

**共情桥：**
- `你不是不会做生意，你是拿实体逻辑硬打线上舆论场。`

**工具化植入：**
- `第3步靠临场发挥很难稳定，需要一套IP+直播承接系统。`
- `这一步不是靠喊口号，是靠标准动作和复盘机制。`

**低压CTA：**
- `要模板，私信我关键词：____。`
- `想看执行清单，我把表格发你。`

### 人物争议文案的植入（5步）

1. 先说共识：这个争议不是个例，很多老板都会遇到类似问题
2. 再说断层：线下经验很强，不代表线上表达就强
3. 再说代价：表达失误会带来信任损耗和获客成本上升
4. 再给动作：先给能立刻执行的3个动作
5. 最后承接：把业务定位成"把动作系统化的工具/方法"

推荐句式：
- 共识句：`这件事不是他一个人的问题，很多老板都会踩这个坑。`
- 断层句：`你线下做得好，不代表线上一开口就能被理解。`
- 动作句：`先做这三步，今天就能开始。`
- 承接句：`如果你想省时间，我有一套现成流程，能帮你把这三步做稳。`

### 植入禁区

- 开头30%出现课程售卖、价格、名额倒计时
- 长段自夸战绩，没有给出任何方法
- 业务段超过全文20%
- "不买就落后/淘汰"式焦虑文案

### 植入位置图（固定）

1. 开场后20%：放人设四步法（短句化）
2. 中段机制解释：放业务"断层+代价"
3. 三条建议第3条：放业务承接
4. 结尾：只做低压CTA

### 植入检查

1. 前40%是否已完成"资历+立场+双边理解"三连？
2. 是否先给出至少2条可执行建议？
3. 业务是否只放在建议第3条或结尾承接区？
4. CTA是否为低压动作？
5. 植入段占比是否<20%？

## 五、语言风格规则

来源：plain-language-rules.md, embedded-live-transcript.md

### 大白话规则

目标：把"新闻热点"翻译成"老板日常经营语言"。

**热点表达规则：**
1. 日期最多保留1-2个关键节点，禁止一段里堆3个以上日期
2. 不念媒体清单，不报链接口播，不做新闻播音腔
3. 事实只回答三件事：发生了什么、为什么跟老板有关、你现在该做什么

**语言规则：**
1. 优先短句，少用并列长句
2. 少抽象词，多场景词：门店、客户、成交、复购、直播间、线索
3. 专业词要翻译：
   - `叙事反转` → `别人本来不信你，后来开始愿意听你讲`
   - `信任承接` → `把看热闹的人变成愿意下单的人`
   - `流量转化` → `把播放量变成客户咨询和成交`

### 逐字稿风格锚点

1. 大白话、短句、并肩式，不端着，不说教
2. 先讲老板现场，再讲方法，不空谈平台术语
3. 每讲一个观点，都给一个老板能代入的场景
4. 语气可犀利，但不能羞辱、恐吓、贴标签

### 核心观点（必须体现在文案里）

1. 不要只追爆款，要追爆单
2. 做短视频不是为了当网红，而是为了让精准客户更快找到你
3. 老板最宝贵的是时间，内容系统必须可复制、可交付、可持续
4. AI 的价值不是炫技，而是降门槛、提效率、稳结果

### 自检清单

1. 外行老板第一次听，能不能复述出你的3个观点？
2. 是否存在"听起来很高级，但落不到动作"的句子？
3. 是否每段都有"老板下一步动作"？

## 六、质量评分标准

来源：peer-benchmark-scorecard.md, rewrite-checklist.md

### 六维主评分（120分）

1. **开场抓取力**（20分）：前3秒是否有强判断？前30秒是否有具体场景？
2. **结构推进力**（20分）：每300-500字是否有推进点？是否存在连续3句命令式语气？
3. **人设可信度**（20分）：前40%是否完成人设预埋链？
4. **业务植入丝滑度**（20分）：植入是否自然？占比是否<20%？
5. **听感与可读性**（20分）：是否像"拆局"而不是"讲课"？
6. **风险与边界控制**（20分）：不散播焦虑、不虚构事实、不夸张承诺、真诚感清晰

### 通过标准

同时满足才算通过：
1. 六维总分 >= 105
2. 单项最低分 >= 16
3. 不触发一票否决项

### 一票否决项

任意命中直接重写：
1. 通过恐吓制造焦虑推进转化
2. 虚构人物、数据、案例或结果
3. 明显夸大效果或做不实承诺
4. 强推销导致真诚感缺失

### 录音改写检查清单（如果是从口播稿改写）

1. 核心观点是否与原文一致
2. 是否出现新增结论
3. 黄金三秒是否抓人
4. 是否只删无效内容
5. 文案是否可直接口播
6. 是否保留用户原有语感

## 七、差异化框架

来源：diff-framework.md, peer-hot-copy-deep-analysis.md

### 你 vs 同行 10维对比（写作时对照优化）

| 维度 | 方法论型老板风格 | 情绪型同行风格 | 优化方向 |
|------|-----------|---------|---------|
| 定位主张 | 偏方法论与系统建设 | 偏立场与情绪带入 | 保留方法论，但前15秒先给态度与冲突 |
| 开场爆点 | 开场理性 | 开场情绪极强 | 开场一句观点+一句反差+一句代入 |
| 情绪抓手 | 稳健讲逻辑 | 强共情+强批判 | 增强"老板心里话"情绪词，避免攻击性 |
| 结构推进 | 知识点驱动 | 戏剧冲突驱动 | 先戏剧后理论，减少前置解释 |
| 证据案例 | 行业规律和抽象逻辑 | 人物事件书籍类比混用 | 每400-600字至少一个案例或类比 |
| 转折升维 | 转折偏平滑 | 反差大记忆点强 | 增加"你以为…其实…"反转句式 |
| 业务种草 | 出现较晚逻辑完整 | 借建议第3条自然导向 | 用6步预埋链 |
| 风险控制 | 合规较好 | 流量强但易踩线 | 保留锋利，去掉贬损与过度标签化 |
| 人设构建 | 专业稳定 | 先情绪后资历 | 前40%完成预埋链 |
| 听感留存 | 逻辑完整偏讲课 | 冲突推进快反转频率高 | 每300-500字设置推进点 |

### 同行热点文案的核心机制（可复制部分）

**注意力机制：**
- 开头先判词，第一句直接给强结论
- 不先讲背景，先让用户站队，再补事实
- 画面化表达，人物动作、朋友圈原话、电影比喻
- 强对立关系，复杂商业问题变成"两个角色两种打法"

**留存机制：**
- 每隔几段推进一次：新事实、新判断、新反转轮替
- "你以为/其实"不断刷新预期
- 理论锚点中段出现
- 从热点升级到方法

**业务转化机制：**
- 不是结尾硬卖，而是全程预埋"能力缺口"
- 顺序：观点→机制→建议→业务成为解法工具
- 业务通常放在第3条建议
- 低压CTA承接线索

## 八、默认业务画像

来源：qing-business-profile.md, embedded-live-transcript.md

### 我是谁

1. 不是讲概念的网红型讲师，是做过长期实战的操盘手
2. 长期服务老板群体，核心关注"能不能拿到客户、能不能成交"
3. 立场：站在老板一侧，给可执行的落地方法

### 我服务谁

1. 传统行业老板、实体门店老板、工厂老板、本地服务老板
2. 典型状态：线下客流下滑，线上不会做，或者做了没结果
3. 真实诉求：不是当网红，不是追虚荣流量，而是稳定获客、稳定成交

### 我解决什么问题

1. 老板没时间：不能每天重度拍摄、剪辑、研究平台规则
2. 老板不会写：文案起不来，内容不成体系
3. 老板怕踩坑：花钱投流、外包代运营、追热点但不转化
4. 老板怕违规：行业敏感词多，内容合规成本高

### 我的方法路径

1. 先定位：围绕生意本身，明确客户画像、卖点、差异化
2. 再模板化：用"既有流量又有销量"的结构模板生成内容
3. 再低门槛执行：用AI完成文案、出镜替代、剪辑提效
4. 再矩阵放大：有效内容改写复用，持续触达

### 目标人群内容偏好

- 要大白话、要结果导向、要可执行，不爱理论空谈

### 默认业务解法（用于植入）

1. 先做业务信息分析，明确定位与差异化
2. 再用模板批量生成"既有流量又有销量"的文案
3. 再用数字人/自动剪辑降低执行难度
4. 再用矩阵分发和复用改写提升触达效率

## 九、表达禁区与风险控制

来源：多个文件交叉提取

### 硬约束（触发任何一条必须改）

1. **禁夸张承诺**：如"躺赚、逆天改命、必赚、马上翻盘、100%结果"
2. **禁恐吓淘汰**：如"不会就等死/必被淘汰"
3. **禁硬逼单**：只做低压引导（关注、评论关键词、私信领取）
4. **禁人身攻击**：不贬损具体人物
5. **禁绝对化标签**：不给人贴死标签
6. **禁虚构事实**：不编造数据、案例、人物

### 风险替换词（强制）

- "原罪" → "天然舆论劣势"
- "肮脏" → "容易被负面解读"
- "打死/殴打" → "单向压制/舆论失衡"
- "爆款"（作为目标时） → "爆单"

### 表达偏好

- 优先"爆单"表述，不把"爆款"当最终目标
- 避免拗口概念和自造词，确保普通老板一遍听懂
- 语气可犀利，但不能羞辱、恐吓、贴标签
